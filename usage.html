LLM API Rotator

🧘 A featherweight, zero-dependency OpenAI-compatible adapter that automatically rotates across free-tier LLM APIs when daily quota is exhausted.

Single file. Zero pip install. Zero database. Zero configuration beyond a JSON list of providers.

Zero dependencies ~500 lines 🏠 Local-only Quota rotation OpenAI format CLI + Server No streaming

✨ Features

🧘 Zero dependenciesPure Python stdlib — urllib + http.server. No pip, no venv, no Docker.
🪶 Single file~500 lines of readable Python. Easy to audit, patch, or embed.
🔄 Auto rotationDetects quota exhaustion (HTTP 402/403/429 + keyword matching) and switches to the next provider transparently.
🖥️ Two modesOne-shot CLI for scripting, or persistent forwarding server for any OpenAI-compatible client.
🔑 Env-safe tokensapi_token_env keeps keys out of committed files.
⏰ TTL cooldownQuota-exhausted providers are skipped for a configurable duration (default 24h).
🏷️ Stable model nameServer mode exposes one fixed model name to clients — backend routing is invisible.

🎯 When to use LLM API Rotator

You're in the right place if:

  • You want to rotate across multiple free-tier APIs (DeepSeek, GLM, Kimi, etc.) with auto-failover when daily quota runs out
  • You don't need streaming — your use case is chat completion, summarization, classification
  • You want dead-simple local deployment — no Docker, no DB, no Redis, no migrations
  • You're running on a low-resource machine — ~10MB RAM, ~0% CPU when idle
  • You want to audit every line of code — 500 lines, pure Python, no compiled extensions
  • You want a single-file embeddable solution — drop llm_api_rotator.py into any project

🏭 When to use LiteLLM

LiteLLM is the better choice if:

  • You need streaming for real-time chat or agentic tool calls
  • You need tool calling / function calling for structured LLM interactions
  • You're spending real money and need cost tracking, budgets, and rate limits
  • You run a multi-tenant service with virtual API keys, user management, and audit logs
  • You need production-grade reliability with Redis-backed cooldowns and Prometheus metrics
  • You deploy on Kubernetes and need health probes and horizontal scaling

In short: LLM API Rotator is a ~500-line stdlib script for personal use with free APIs. LiteLLM is a 100K+ line enterprise proxy. Pick the one that matches your complexity budget.

🚀 Quick Start

1. Create providers.json

{
  "providers": [
    {
      "name": "deepseek-free",
      "base_url": "https://api.deepseek.com/v1",
      "api_token_env": "DEEPSEEK_API_KEY",
      "model": "deepseek-chat"
    },
    {
      "name": "glm-free",
      "base_url": "https://open.bigmodel.cn/api/paas/v4",
      "api_token_env": "GLM_API_KEY",
      "model": "glm-4-flash"
    }
  ]
}

Set tokens in your environment:

export DEEPSEEK_API_KEY="sk-..."
export GLM_API_KEY="..."

2. One-shot prompt

python3 scripts/llm_api_rotator.py \
  --config providers.json \
  --prompt "Write a haiku about LLMs"

Stdout gets the reply, stderr gets used_provider=deepseek-free.

3. Server mode

python3 scripts/llm_api_rotator.py \
  --config providers.json \
  --serve \
  --host 127.0.0.1 \
  --port 8000 \
  --local-model local-rotator

Point any OpenAI client at http://127.0.0.1:8000/v1 with model local-rotator and the printed API key.

🔄 How Rotation Works

  1. Providers are tried in order as listed in providers.json.
  2. Quota is detected reactively — the script sends the request and watches for:
    • Status codes: HTTP 402, 403, 429
    • Error keywords: quota, rate limit, insufficient_quota, billing, credit, free tier, daily limit, tokens exhausted
  3. On quota error → marks provider exhausted for TTL (default 24h), retries same request with the next provider.
  4. If all providers fail, an error is returned.

Server mode lifecycle

Client sends:    model = local-rotator
Server tries:    provider-a / backend-model-a
Quota error:     provider-a is skipped (24h cooldown)
Server retries:  provider-b / backend-model-b
Client sees:     model = local-rotator (unchanged)

⚠️ Limitations

  • No streaming — stream: true is silently converted to false
  • No tool/function calling — pure chat completions only
  • No token pre-checking — quota is only detected when a request actually fails
  • Single stable model name — server mode exposes one model to all clients
  • In-memory state — restarting resets all cooldowns (fine for daily-reset free APIs)

🔒 Security

Never commit API tokens. Use api_token_env in your config and export the real key as an environment variable.

The local API key is only for client-to-server auth — upstream provider keys still come from the config file. The local key changes every server start unless you set --local-api-key or --local-api-key-env.
